酥师傅的毕业寄

发表于 2023-06-01 分类于 酥饼周报 本文字数： 3.3k 阅读时长 ≈ 12 分钟

论文的致谢草草了事的，是一个非常标准的致谢——其实我很想搞事写一个“感谢国家”之类的东西。但毕竟，个人水平不足以支撑我的装B，所以还是中规中矩的写吧。

从走进实验室跟老板坐下第一次聊天到现在，已经在这里待了7年零8个月了。

曾经入学研究生时被当作是老生，令老板产生了“怎么今年没有新生”的错觉。

在实验室经历了大搬家、再次大搬家和小搬家等一系列非磕盐活动，动手能力得到了极大的锻炼。

曾在短短一年间掌握了实验室几乎所有现有设备的搭建和维修方法，并且可以闭着眼指出屋里面每个抽屉、每个柜子放了什么东西。

这直接导致我一度认为自己是个实验员而不是研究生。

自我定位的一些偏差导致了学术成果一般般，毕业论文也是混混了事，但做实验的寄术倒能算得上有两把刷子。

在实验室待得太久，即使是个人数不多的小课题组，我也算是送走了大量的毕业生。终于熬到今年，轮到自己送自己。

既然，学术上可以说一无所成，那么值得分享的，就是一点寄语和一些微不足道的寄术了。

写论文的煎熬

从去年大约9月多就开始写了，那个时候的想法是先写成分项报告，然后再攒起来。但由于各种杂事，分项报告也不过是写了个开头，然后就直接上手写大论文了。前前后后是写了小半年，写成了一份能够预答辩的初稿。

期间一度认为自己已经写不完了，要不然延毕吧。但等写到正文纯文字7W字的时候，看到了希望，感觉到必须要争取当前毕业（总字数要求8W）。

最终勉强写好一篇看得过去的论文，修修改改两个月，走到最后这步毕业答辩。

——分享一些毕业的痛苦与经验

论文的撰写工具——初次接触Latex

写论文的时候用了Latex，第一次用。主要原因就是不想最后的时候改格式改成狗。

从论文撰写的初期开始，其实试用了WPS，MS office和Latex。体验了不同软件，并最终选择了Latex作为论文排版的工具（实际上也是边写边排）。

下面是各个软件值得劝退的要点（个人认为软件的优势并不能成为我选择它的理由，但其短板必然是我拒绝的理由）：

WPS：一款磕盐不友好的软件

稍有优势的点可能就剩下网盘在国内还方便一点，但MS Office有学校账户啊！

WPS写正文的最大难题就是打字还离不开鼠标，诸多不能“自动修正”或者“自动补齐”，这种情况其实比较影响大段落的打字：

  • 数学公式：不能像MS Office一样，无法识别你输入Latex格式，每次输入角标、加入分式、输入希腊字符时都要手动去点击公式菜单。
  • 自动修正：无法自定义自动修正，因为经常要使用“pH”，”MPa”，“°C”，“μm”等文字，字母的大小写要禁止自动修正，同时我希望我输入“\degc”就自动转换为“°C”这样。特别是μm，一般都要用Time New Roma字体，但是“μ”如果用输入法“miu”打出来的话就会变成宋体，格式不统一。那么设置一个自动修正“\um”为“μm”，就可以实现切换输入法来输入正确字体的μm。

制图方面，WPS的制图也令人难以接受

  • PPT制图，将微观图片或者形状、文字等组合好后，如果复制再粘贴为图片，则分辨率会大幅下降，图片细节大量失真（例如EBSD图片中，晶界出现阴影、晶界的实线与晶内颜色的边界并不重合的问题）。
  • 图片另存为选项同样会导致保存的图片模糊，这样的情况致使我很难接受，根本不能做出论文格式需求的图片。
  • WPS自带的图片压缩也很抽象，例如我要在PPT里面制作带有标尺的微观形貌图时，插入一个标尺”50 μm“，图片压缩后这个“μm”就会变成诸多非UNICODE程序一样的”衸“。

总之WPS在打字和制图这两大方面的不可接受的BUG，让我放弃了这款软件——一些对国产软件恨铁不成钢的情感，我甚至多次发截图、意见反馈给金山，但他们完全没有回复，三年以来这些问题没有一个得到解决或优化，所以，也就彻底放弃了这个软件。

MS Office：解决以上的所有问题，但排版很累（其实也不是很累）

  • 最大的一个问题就是图文超级多的情况下的卡顿问题，但再怎么说也比WPS好多了；
  • 用WPS或者MS office调格式对我而言其实没有任何麻烦的地方，因为我自己写东西的习惯是先搞定文档模板再下手写东西，而且全部的图文必须是我手打进去，除了从自己写过的文档里面复制粘贴。所以格式基本不会错乱；
  • 超长的几十万字的论文的排版就有些困难了，主要是烦躁，特别是更换图片的时候，删除或者插入一个图片，后面就会卡成PPT。但其实是分章节分文档的话，可能也不会卡。之后可能会尝试使用以下Word的分文档和合并文档的功能吧；
  • 最大的困难可能就是学校没有提供一个标准的Word模板，然后公式的编号不能很简单的实现【虽然我也用制表符的方式实现了编号，但总感觉很别扭】。

Latex：长篇大论文的救星，但短篇论文很鸡肋

  • 最开始用的overleaf，在线编辑无需自己配置环境，学校也购买了官方服务，服务器在学校手里，所以既能体验到付费的共享功能，也能确保服务器相对稳定；
  • BUT！随着我的文档中图片越来越多，编译时间一度超过了十分钟，最终导致编译时间超时；
    • 只能使用快速draft模式，不加载图片。但往往不加载图片的情况下我很难判断我的图文效果；
    • 其实还有个问题就是，帮我改论文的同志们并不是很能操作这个东西——那么在线编辑、共享文档的意义也不是很大了；
  • 换回了离线版，VS code + MikTex，对于我个人而言十分舒适，根本不需要再考虑格式是否正确的问题。本就不足的脑容量可以腾出很大一部分用来思考如何去优化我那十分LOW的毕设内容；
    • 离线的情况下没有类似的draft模式——也就是不加载图片的模式，这导致我自己想要快速预览的情况也很不方便，应该有这个类似的功能，但我是在太菜，没找到解决方案；
    • 相比在线版，可以实时预览公式、图片的效果，这对于我这个编辑公式还很不熟练的人而言，比较友好了。这种情况下不能快速预览全文似乎也不是非常重要了。
    • 要自己配置环境，头两次配置的比较失败，不知哪里出了问题，总是编译不出来。最后一次成功了，我把全部文档放在了学校OneDrive的网盘里，然后实验室台式机和自己的笔记本都能编译。

  • 最终论文从2W不到的字数写到带参考文献10W+的过程都是在Latex上实现的，格式一边过（其实有几个我自己漏写的单位，忘记使用siunit语法）
  • 10页WORD以内的东西我不会考虑Latex了，毕竟，要不是大论文格式要求多，我也不会想一直用Latex。
  • 喔，还有一个非常不友好的问题：Noteexpress对bibtex格式的支持太差，一些会议论文、标准、网站、书中章节输出的bibtex格式并不正确，搞得我还得自己重新设定输出模板【前几天更新的版本也没有改进这个问题】。呜呜呜…

作图工具PowerPoint——一款造模型的利器

年轻时用PPT做PPT，现在用PPT做模型

为什么不用Adobe全家桶：

早年我其实是有Adobe全家桶操作经验的，做宣传岗的经历让我熟练掌握了PS，AI的操作，毕竟是某年研究生新生手册的核心排版人员（苦工）。

但，学术研究、撰写论文，讲究的一个简单实用：PhotoShop华而不实，PowerPoint强劲有力！

确实，绘制一些示意图、流程图什么的，Adobe全家桶有些大材小用的意思，主要是不够轻快。

借助PPT，基本上是完成了论文的全部非数据图片的绘制工作：

  • 拼图：主要是拍摄的微观形貌、表征结果的组合，以及标尺的绘制。
  • 示意图：就如上面两张图一样，一些是机理机制的示意图，另一些是加工过程或测试方法的示意图

抠图也基本上是PPT搞定的，毕竟PPT做完了直接就拿来可以做答辩PPT了。

相比Adobe全家桶的强大功能，PPT在画图上的功能简单却足够用，我的写论文的文件夹中往往存在一个“制图.pptx”的文档，这里面每页都是一个示意图或者一幅组合图，随时可以快速修改。

修改好的形状直接选中复制，然后进入Word中粘贴为图片。

也感谢MS office在粘贴为图片的时候没有像WPS那样狠狠地压缩，让我能够得到一些相对高清的图片。

一些注意要点：

  • 作图的可读性：其实主要是图中的文字大小，一般学位论文正文的宽度也只有14~15cm，我的习惯是专门准备一个PPT模板，宽度就是14cm，高度10cm（一般插入的图片多是横向的版式，这个尺寸基本够用），制作示意图或者流程图的时候保证内容与PPT的页面大小相近；
  • 文字尺寸设置：论文正文一般就是五号或者小四，因此PPT作图也要用五号或者小四：10.5，12磅，最小的字体也就是8号（这种情况其实已经不太便于阅读了，但图中内容很多，文字区域较小的情况下也可以用到小号字）。
  • 其他图片：数据图表都是用OriginLab做的，选中多个图表，”Ctrl + G“就可以进入图片导出的窗口，我自己自定义了一个输出7cm宽度、300DPI、PNG格式的模板，这样非常便于我导出尺寸一致的图片，7cm是适合双栏图表的，OriginLab的字体28~36磅的情况下能保证插入到正文里面的视觉效果较好。
  • 裁剪边缘：作为一个轻度强迫症患者，有些无法接受生成的图片残留白边。可能与文本框的边距之类的有关，当你设计的形状或者组合图中存在文本框时，且文本框在整个图形的边缘，那么复制出或另存为的图片就会存在一个白边。

建站的理由

其实，新建这篇文章的时候是想写一些临别感悟，最终又变成了一篇如何打字和画图的经验分享，可能我磕盐这么多年感悟就是如此吧。

——分享一点点微不足道的磕盐经验（主要是技能点在了维修上）

在临近毕业的时候开始了瞎搞，建了这个小小的个人站，主要想要分享一些过去十年积累下的各种知识。

过去十年，甚至十几年的人生其实都是大量的踩坑与偶然的幸运中度过的，幸运的事情值得分享，但快乐这种事情，也不是人人都能共情。但踩坑一定值得分享，毕竟之后的人能够少踩一个就稳赚不赔了。
